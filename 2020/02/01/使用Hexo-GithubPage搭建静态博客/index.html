使用Hexo+GithubPage搭建静态博客

发表于 2020-02-01

Hexo是一个 快速、简洁且高效的博客框架 。对于大多数学生而言，没钱没技术，成本低、方便快捷成为了学生们的第一需求。所以方便的Hexo和免费的GithubPage成为了许多学生们的选择。接下来，本文就分享一下小石子搭建博客的经验。

//我的电脑系统为Windows10，装有node.js，git。

②配置Hexo

安装完Hexo，在你想建站的文件夹里执行以下指令初始化Hexo

创建完成后，应有以下文件

_config.yml

网站的 配置 信息，您可以在此配置大部分的参数。

package.json

应用程序的信息。EJS, Stylus 和 Markdown renderer 已默认安装，您可以自由移除。

scaffolds

模版 文件夹。当您新建文章时，Hexo 会根据 scaffold 来建立文件。

Hexo的模板是指在新建的文章文件中默认填充的内容。例如，如果您修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。

source

资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes

主题 文件夹。Hexo 会根据主题来生成静态页面。

创建完成后，在 _config.yml中可以修改大部分的Hexo配置。

配置的具体部分参考官方文文档

第三步：将Hexo部署到GithubPage

首先你要有一个github账户，然后新建一个按 yourname.github.io 命名格式的仓库。Github会帮你开启Github Page。通过 http://yourname.github.io 可访问你的Github Page。
